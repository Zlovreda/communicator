# Contributor License Agreement (CLA) для проекта "Ком-Ком"

## Что такое CLA?

CLA (Contributor License Agreement) - это юридическое соглашение между участником и владельцами проекта, которое устанавливает условия для принятия вкладов (кода, документации и т.д.).

## Зачем нужен CLA?

1. **Защита проекта**: Обеспечивает, что все вклады имеют четкие лицензионные условия
2. **Защита участников**: Четко определяет права и обязанности
3. **Будущая совместимость**: Позволяет изменить лицензию проекта при необходимости
4. **Юридическая ясность**: Упрощает коммерческое использование и дистрибуцию

## Как подписать CLA?

### Для физических лиц:
1. Создайте Pull Request с вашим вкладом
2. В комментарии к Pull Request напишите: Я, [Ваше полное имя], согласен с условиями CLA
3. 3. Дождитесь проверки и принятия

### Для организаций:
1. Уполномоченный представитель должен отправить письмо на [PKopchenov@gmail.com]
2. В письме указать полные реквизиты организации
3. После подтверждения, сотрудники организации могут создавать Pull Request

## Вопросы и ответы

### Q: Нужно ли подписывать CLA для каждого Pull Request?
A: Нет, подписание действует для всех будущих вкладов от этого участника/организации.

### Q: Можно ли вносить мелкие исправления без CLA?
A: Незначительные правки (опечатки, форматирование) обычно не требуют CLA, но для значительных изменений CLA обязателен.

### Q: Что если я не хочу подписывать CLA?
A: Вы можете использовать проект в соответствии с лицензией Apache 2.0, но не сможете вносить вклады в основную кодобазу.

### Q: CLA передает права на мой код?
A: Нет, CLA только предоставляет лицензию на использование вашего кода в проекте на условиях Apache 2.0. Вы сохраняете авторские права.
