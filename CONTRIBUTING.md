## CLA (Contributor License Agreement)

Перед тем как внести значительный вклад, пожалуйста, подпишите CLA:

- **Физические лица**: Прочтите [INDIVIDUAL-CLA.md](cla/INDIVIDUAL-CLA.md) и подтвердите согласие в комментарии к Pull Request
- **Организации**: Прочтите [CORPORATE-CLA.md](cla/CORPORATE-CLA.md) и свяжитесь с нами по email

Без подписанного CLA мы не сможем принять ваш Pull Request.

## Процесс внесения вкладов

1. **Обсудите изменение**: Создайте Issue или обсудите в Discussions
2. **Подпишите CLA**: Как описано выше
3. **Создайте ветку**: `git checkout -b feature/amazing-feature`
4. **Внесите изменения**: Соблюдайте стиль кода проекта
5. **Добавьте тесты**: Если это возможно и уместно
6. **Зафиксируйте изменения**: `git commit -m 'Add amazing feature'`
7. **Отправьте изменения**: `git push origin feature/amazing-feature`
8. **Создайте Pull Request**: Укажите номер Issue, если есть
9. **Дождитесь проверки**: Может потребоваться несколько итераций
10. **Получите аппрув**: После approval изменения будут смержены
